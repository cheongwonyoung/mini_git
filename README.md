# 협업 규칙

## Commit 메세지

- 제목과 본문을 빈 행으로 구분합니다.
- 제목을 50글자 이내로 제한합니다.
- 제목의 첫 글자는 대문자로 작성합니다.
- 제목의 끝에는 마침표를 넣지 않습니다.
- 제목은 명령문으로! 과거형을 사용하지 않습니다.
- 본문의 각 행은 72글자 내로 제한합니다.
- 어떻게 보다는 무엇과 왜를 설명합니다.

### Ex ) git commit -m “[Feat]: 더하기 기능 추가”

- [Feat] : 기능 추가
- [Fix] : 기능 수정
